Retrieve the current state of a single generation by its ID. Use this endpoint to check whether a generation is still processing, has completed with a downloadable video, or has failed with an error. You can call this endpoint repeatedly to poll for status changes after [creating a generation](/api-reference/generations/create).

## Endpoint

```
GET https://api.techblazing.com/v1/generations/{id}
```

## Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the generation to retrieve. Generation IDs are returned in the response when you [create a generation](/api-reference/generations/create) and follow the format `gen_<ulid>`.
</ParamField>

## Response

Returns the full [Generation object](#generation-object) reflecting the current state of the job.

```json
{
  "id": "gen_01H9K7XVMZQ3P4R5T6Y8W",
  "object": "generation",
  "status": "completed",
  "prompt": "A cinematic shot of a mountain at golden hour, wide angle, dramatic lighting",
  "style": "cinematic",
  "aspect_ratio": "16:9",
  "duration": 15,
  "quality": "hd",
  "input_asset_id": null,
  "script": null,
  "output_url": "https://cdn.techblazing.com/outputs/gen_01H9K7XVMZQ3P4R5T6Y8W.mp4?token=abc123&expires=1720000000",
  "error": null,
  "created_at": "2024-07-01T14:23:00Z",
  "completed_at": "2024-07-01T14:24:12Z"
}
```

### Generation Object

| Field            | Type          | Description                                                                                          |
|------------------|---------------|------------------------------------------------------------------------------------------------------|
| `id`             | string        | Unique identifier for the generation.                                                                |
| `object`         | string        | Always `"generation"`.                                                                               |
| `status`         | string        | Current status: `queued`, `processing`, `completed`, or `failed`.                                   |
| `prompt`         | string        | The text prompt used to generate the video.                                                          |
| `style`          | string        | The visual style applied: `realistic`, `cinematic`, `animated`, or `illustrated`.                   |
| `aspect_ratio`   | string        | Output aspect ratio: `16:9`, `9:16`, or `1:1`.                                                      |
| `duration`       | integer       | Video length in seconds.                                                                             |
| `quality`        | string        | Output quality: `standard` or `hd`.                                                                 |
| `input_asset_id` | string\|null  | ID of the reference image asset, if one was provided.                                                |
| `script`         | string\|null  | Narration script used for voiceover, if one was provided.                                            |
| `output_url`     | string\|null  | Pre-signed URL to download the completed video. `null` until status is `completed`.                 |
| `error`          | object\|null  | Error details when status is `failed`, otherwise `null`.                                             |
| `created_at`     | string        | ISO 8601 timestamp of when the generation was created.                                               |
| `completed_at`   | string\|null  | ISO 8601 timestamp of when the generation finished. `null` until the job reaches a terminal state.  |

## Code Examples

<CodeGroup>

```bash curl
curl --request GET \
  --url https://api.techblazing.com/v1/generations/gen_01H9K7XVMZQ3P4R5T6Y8W \
  --header 'Authorization: Bearer <API_KEY>'
```

```javascript Node.js
const generationId = 'gen_01H9K7XVMZQ3P4R5T6Y8W';

const response = await fetch(
  `https://api.techblazing.com/v1/generations/${generationId}`,
  {
    headers: {
      'Authorization': 'Bearer <API_KEY>',
    },
  }
);

const generation = await response.json();

if (generation.status === 'completed') {
  console.log('Video ready:', generation.output_url);
} else if (generation.status === 'failed') {
  console.error('Generation failed:', generation.error);
} else {
  console.log('Still processing, current status:', generation.status);
}
```

```python Python
import httpx

generation_id = 'gen_01H9K7XVMZQ3P4R5T6Y8W'

response = httpx.get(
    f'https://api.techblazing.com/v1/generations/{generation_id}',
    headers={'Authorization': 'Bearer <API_KEY>'},
)

generation = response.json()

if generation['status'] == 'completed':
    print('Video ready:', generation['output_url'])
elif generation['status'] == 'failed':
    print('Generation failed:', generation['error'])
else:
    print('Still processing, current status:', generation['status'])
```

</CodeGroup>

<Note>
  When polling for status changes, wait at least **5–10 seconds** between requests. Polling more frequently than once every 5 seconds is unlikely to return a different result and may cause your requests to be rate-limited. For a lower-latency alternative, provide a `webhook_url` when [creating your generation](/api-reference/generations/create) to receive a push notification the moment the job completes.
</Note>

<Warning>
  The `output_url` is a pre-signed URL that **expires 24 hours** after the generation completes. Download and store the video file in your own storage before the URL expires — after that point you will not be able to retrieve it through this endpoint.
</Warning>
