Cancel a generation that is currently `queued` or `processing` ‚Äî for example, if you submitted a job with incorrect parameters, want to free up processing capacity, or no longer need the output. Once cancelled, the generation stops immediately and will not produce an output video. You cannot reverse a cancellation, but you can always [submit a new generation](/api-reference/generations/create) with updated parameters.

## Endpoint

```
POST https://api.techblazing.com/v1/generations/{id}/cancel
```

## Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the generation to cancel. The generation must currently have a status of `queued` or `processing`.
</ParamField>

## Response

Returns the updated [Generation object](/api-reference/generations/retrieve) with `status` set to `"cancelled"`.


## Code Examples

<CodeGroup>

```bash curl
curl --request POST \
  --url https://api.techblazing.com/v1/generations/gen_01H9K7XVMZQ3P4R5T6Y8W/cancel \
  --header 'Authorization: Bearer <API_KEY>'
```

```javascript Node.js
const generationId = 'gen_01H9K7XVMZQ3P4R5T6Y8W';

const response = await fetch(
  `https://api.techblazing.com/v1/generations/${generationId}/cancel`,
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer <API_KEY>',
    },
  }
);

const generation = await response.json();
console.log(generation.status); // cancelled
```

```python Python
import httpx

generation_id = 'gen_01H9K7XVMZQ3P4R5T6Y8W'

response = httpx.post(
    f'https://api.techblazing.com/v1/generations/{generation_id}/cancel',
    headers={'Authorization': 'Bearer <API_KEY>'},
)

generation = response.json()
print(generation['status'])  # cancelled
```

</CodeGroup>

<Note>
  You can only cancel a generation with a status of `queued` or `processing`. Attempting to cancel a generation that has already reached a terminal state ‚Äî `completed`, `failed`, or `cancelled` ‚Äî returns a **409 Conflict** error. Check the generation's current status with the [Retrieve endpoint](/api-reference/generations/retrieve) before cancelling if you are unsure.
</Note>

<Info>
  If you cancel a generation while it is still `queued` (i.e., processing has not yet started), any credits consumed by the request are automatically refunded to your account. Cancellations of `processing` jobs are not eligible for a credit refund, as compute resources will have already been allocated. Refunded credits appear in your account balance within a few minutes.
</Info>
