The TechBlazing REST API gives you programmatic access to the full TechBlazing platform — generate AI-powered videos, manage your media assets, and configure webhooks to receive real-time event notifications. This reference covers everything you need to make your first request and integrate TechBlazing into your application.

## Base URL

All API requests are made over HTTPS to the following base URL. Any request made over plain HTTP will be rejected.

```
https://api.techblazing.com/v1
```

The version identifier (`v1`) is part of the URL path. See [Versioning](#versioning) for details.

## Authentication

TechBlazing uses API keys to authenticate requests. Include your API key as a Bearer token in the `Authorization` header of every request. You can generate and manage your API keys from the [TechBlazing dashboard](https://app.techblazing.com/settings/api-keys).

```http
Authorization: Bearer <API_KEY>
```

<Warning>
  Keep your API keys secret. Never expose them in client-side code, public repositories, or logs. If a key is compromised, rotate it immediately from your dashboard.
</Warning>

Requests without a valid API key return a `401 Unauthorized` response. Requests with a key that lacks the required permissions return a `403 Forbidden` response.

## Request Format

Send request bodies as JSON and set the `Content-Type` header accordingly on all `POST`, `PUT`, and `PATCH` requests.

```http
Content-Type: application/json
```

Query parameters are used for filtering, pagination, and other read-only options on `GET` requests.

## Response Format

All responses are returned as JSON. Successful responses for single-resource operations return the resource object directly at the top level:

```json
{
  "id": "gen_01H9K7XVMZQ3P4R5T6Y8W",
  "object": "generation",
  "status": "completed",
  "created_at": "2024-07-01T14:23:00Z"
}
```

List endpoints return a `data` array alongside pagination metadata:

```json
{
  "data": [...],
  "has_more": true,
  "next_cursor": "gen_01H9K2ABCD1234567890X"
}
```

Timestamps are always returned in [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) format (UTC).

## Versioning

The current API version is **v1**, reflected in the base URL path. When TechBlazing introduces breaking changes, a new version will be released at a new path (e.g., `/v2`). Non-breaking additions — such as new fields on existing objects or new optional request parameters — may be added to the current version without notice, so build your integration to gracefully handle unknown fields.

## Example Request & Response

The following example generates a new video from a text prompt. Replace `YOUR_API_KEY` with your actual key.

<CodeGroup>

```bash curl
curl --request POST \
  --url https://api.techblazing.com/v1/generations \
  --header "Authorization: Bearer YOUR_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{
    "prompt": "A cinematic drone shot of a mountain range at golden hour",
    "duration": 10,
    "quality": "hd"
  }'
```

```javascript Node.js
const response = await fetch("https://api.techblazing.com/v1/generations", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    prompt: "A cinematic drone shot of a mountain range at golden hour",
    duration: 10,
    quality: "hd",
  }),
});

const generation = await response.json();
console.log(generation.id); // gen_01H9K7XVMZQ3P4R5T6Y8W
```

```python Python
import httpx

response = httpx.post(
    "https://api.techblazing.com/v1/generations",
    headers={
        "Authorization": "Bearer YOUR_API_KEY",
        "Content-Type": "application/json",
    },
    json={
        "prompt": "A cinematic drone shot of a mountain range at golden hour",
        "duration": 10,
        "quality": "hd",
    },
)

generation = response.json()
print(generation["id"])  # gen_01H9K7XVMZQ3P4R5T6Y8W
```

</CodeGroup>

**Response**

```json
{
  "id": "gen_01H9K7XVMZQ3P4R5T6Y8W",

  "duration": 10,
  "quality": "hd",
  "output_url": null,
  "created_at": "2024-05-01T12:00:00Z",
  "completed_at": null

```
