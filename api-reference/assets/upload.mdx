Upload an image asset to TechBlazing so you can reference it in video generation requests. Once uploaded, the returned asset `id` can be passed as `input_asset_id` when creating a generation, giving the model a visual anchor — such as a product shot, brand logo, or character — to incorporate into the output video.

## Endpoint

```
POST https://api.techblazing.com/v1/assets
```

Requests must use `multipart/form-data` encoding.

## Request

<ParamField body="file" type="file" required>
  The image file to upload. Accepted formats: `image/jpeg`, `image/png`, `image/webp`. Maximum file size is **10 MB**.
</ParamField>

<ParamField body="filename" type="string">
  A custom filename to label the asset for your own reference. If omitted, TechBlazing uses the original filename from the upload.
</ParamField>

## Response

Returns an [Asset object](#asset-object) on success.

### Asset object

| Field          | Type    | Description                                                  |
| -------------- | ------- | ------------------------------------------------------------ |
| `id`           | string  | Unique identifier for the asset, prefixed with `ast_`.       |
| `object`       | string  | Always `"asset"`.                                            |
| `filename`     | string  | The stored filename of the uploaded asset.                   |
| `content_type` | string  | MIME type of the uploaded file.                              |
| `size`         | integer | File size in bytes.                                          |
| `url`          | string  | CDN URL where the asset can be accessed.                     |
| `created_at`   | string  | ISO 8601 timestamp of when the asset was uploaded.           |

```json Example response
{
  "id": "ast_01H9K7XVMZQ3P4R5T6Y8W",
  "object": "asset",
  "filename": "product-shot.jpg",
  "content_type": "image/jpeg",
  "size": 2048576,
  "url": "https://cdn.techblazing.com/assets/ast_01H9K7XVMZQ3P4R5T6Y8W.jpg",
  "created_at": "2024-07-01T14:20:00Z"
}
```

## Code examples

<CodeGroup>

```bash curl
curl --request POST \
  --url https://api.techblazing.com/v1/assets \
  --header "Authorization: Bearer $TECHBLAZING_API_KEY" \
  --form "file=@/path/to/product-shot.jpg" \
  --form "filename=product-shot.jpg"
```

```javascript Node.js
import fs from "fs";
import FormData from "form-data";
import fetch from "node-fetch";

const form = new FormData();
form.append("file", fs.createReadStream("/path/to/product-shot.jpg"));
form.append("filename", "product-shot.jpg");

const response = await fetch("https://api.techblazing.com/v1/assets", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.TECHBLAZING_API_KEY}`,
    ...form.getHeaders(),
  },
  body: form,
});

const asset = await response.json();
console.log(asset.id); // ast_01H9K7XVMZQ3P4R5T6Y8W
```

```python Python
import os
import requests

with open("/path/to/product-shot.jpg", "rb") as f:
    response = requests.post(
        "https://api.techblazing.com/v1/assets",
        headers={"Authorization": f"Bearer {os.environ['TECHBLAZING_API_KEY']}"},
        files={"file": ("product-shot.jpg", f, "image/jpeg")},
        data={"filename": "product-shot.jpg"},
    )

asset = response.json()
print(asset["id"])  # ast_01H9K7XVMZQ3P4R5T6Y8W
```

</CodeGroup>

<Note>
  Images must be at least **512 × 512 pixels**. Uploads that fall below this minimum will be rejected with a `422 Unprocessable Entity` error. There is no enforced maximum resolution, but very large images are automatically downsampled before being used as generation references.
</Note>

<Tip>
  After a successful upload, pass the returned `id` as `input_asset_id` in your generation request to use this image as a visual reference. For example: `"input_asset_id": "ast_01H9K7XVMZQ3P4R5T6Y8W"`.
</Tip>
